# Testing Harness for Raspberry Pi 5 Images

This project provides an automated testing harness for Raspberry Pi OS images (like those generated by `pi-gen` or `rpi-image-gen`). It deploys the image over the network using PXE boot (TFTP + NFS) to a Raspberry Pi 5.

## Prerequisites

For this harness to work, the host machine running the tests must be connected to the same network as the Raspberry Pi 5, or directly via an Ethernet cable. 

The Raspberry Pi 5 must have its bootloader configured to attempt Network Boot (this is the default fallback if no SD Card or USB drive is inserted).

### Required Packages
You'll need `dnsmasq`, `nfs-kernel-server`, `kpartx`, and `python3`.

## Architecture

1. **Image Preparation**: The harness script takes a `.img` file, maps its partitions using `kpartx`, and mounts them locally.
2. **Network Boot Config**: The script copies the `boot` files to a TFTP directory and the `root` filesystem to an NFS export.
3. **Patching**: It modifies `cmdline.txt` in the boot files to instruct the Linux kernel to mount its root filesystem over NFS instead of from an SD card. It also patches `/etc/fstab` to prevent the Pi from trying to mount local partitions.
4. **Booting**: The `dnsmasq` server responds to the Pi's PXE request, sends the boot files via TFTP, and the Pi boots into the provisioned NFS root.
5. **Testing**: The harness waits for the Pi to become available via SSH, runs predefined tests, and reports the results.
